Montien Tech Terminal

Web SSH Client แบบเรโทร 16-Bit รองรับภาษาไทย, Multi-Tab, Auto-Reconnect และ Pixel Art Mascot — ออกแบบมาเพื่อใช้งาน Claude Code และ CLI ผ่านเบราว์เซอร์

Node.js + Express WebSocket ssh2 xterm.js v5 PWA Ready MIT License
01 — ภาพรวมโปรเจค

คืออะไร?

Montien Tech Terminal คือ SSH Client ที่รันบน Node.js และเปิดผ่านเบราว์เซอร์ได้ทันที ไม่ต้องติดตั้งซอฟต์แวร์เพิ่มเติม ออกแบบมาเพื่อใช้งาน Claude Code CLI และ เครื่องมือ interactive บน Linux Server โดยเฉพาะ

รองรับการพิมพ์คำสั่งภาษาไทยผ่านช่อง input พิเศษ ที่หลีกเลี่ยงปัญหา IME ของ browser ใน xterm.js

ทำไมต้องใช้?

  • 🌐เข้าได้จากทุกอุปกรณ์ที่มีเบราว์เซอร์ — มือถือ, แท็บเล็ต, PC
  • 🇹🇭พิมพ์ภาษาไทยส่งคำสั่งไปยัง server ได้จริง
  • 🔁Auto-Reconnect ถ้าหลุดจะพยายามต่อใหม่อัตโนมัติ
  • 📑เปิดหลาย Tab เชื่อมต่อหลาย server พร้อมกัน
  • 📦รองรับการติดตั้งเป็นPWA ใช้งานได้แบบ native app

Technology Stack

  • 🟢Node.js — runtime หลัก
  • 📦express — serve static files
  • 🔌ws — WebSocket server
  • 🔐ssh2 — SSH client library
  • 💻xterm.js v5 — terminal emulator in browser
  • 🔧xterm-addon-fit — auto-resize terminal
04 — การติดตั้งและเริ่มใช้งาน

Requirements

  • ✅Node.js v18 ขึ้นไป
  • ✅npm (มากับ Node.js)
  • ✅เบราว์เซอร์ modern — Chrome, Firefox, Edge, Safari
  • ✅เครื่อง target ต้องเปิด SSH (port 22 หรือกำหนดเอง)

ขั้นตอนติดตั้ง

  1. Clone หรือ Download โปรเจค 
    git clone <repo-url> ssh_terminal
cd ssh_terminal
  2. ติดตั้ง Dependencies 
    npm install

    จะติดตั้ง: express, ws, ssh2

  3. เริ่มเซิร์ฟเวอร์ 
    npm start
# หรือ
node server.js

    จะขึ้น: Web SSH Client server is running on http://localhost:3000

  4. เปิดเบราว์เซอร์ 
    http://localhost:3000
💡 Tip: ต้องการเปลี่ยน port ให้ตั้ง environment variable: 
PORT=8080 node server.js
ℹ PWA Installation: เปิดเว็บใน Chrome แล้วกด "Install App" ที่ address bar เพื่อใช้งานเหมือน native app (icon บน desktop/homescreen)

  • 🖥รองรับ IP address และ hostname (เช่น server.example.com)
  • 🔑ยืนยันตัวตนด้วย password (กดปุ่มตาเพื่อดู/ซ่อน)
  • 🔧กำหนด port SSH ได้ (default: 22)
  • 💾ระบบจำ host/username ล่าสุดไว้ใน localStorage อัตโนมัติ

📑 Multi-Tab — เปิดหลาย Session พร้อมกัน

กดปุ่ม + ที่ tab bar เพื่อเปิด tab ใหม่ แต่ละ tab คือ SSH session แยกอิสระ

root@prod-server
ubuntu@staging
admin@db-01
＋
root@prod-server:~$ # active session
  • 🔌แต่ละ tab มี WebSocket connection แยกกัน — ไม่รบกวนกัน
  • 🏷Tab title เปลี่ยนเป็น username@host เมื่อเชื่อมต่อสำเร็จ
  • ✕ปิด tab ได้ด้วยปุ่ม ✕ (tab สุดท้ายจะ disconnect แต่ไม่ถูกลบ)
  • ⟳Tab ที่กำลัง reconnect จะแสดง [⟳] username@host

🔁 Auto-Reconnect — ต่อใหม่อัตโนมัติ

หากการเชื่อมต่อขาดหาย ระบบจะพยายาม reconnect อัตโนมัติ สูงสุด 5 ครั้ง

root@server:~$ top
... output ...
 
⚡ การเชื่อมต่อขาดหาย — reconnect ครั้งที่ 1/5 ใน 2s...
⚡ การเชื่อมต่อขาดหาย — reconnect ครั้งที่ 2/5 ใน 4s...
✓ เชื่อมต่อใหม่สำเร็จ!
  • ⏱Delay เพิ่มแบบ exponential backoff: 2s → 4s → 8s → 16s → 30s
  • 5️⃣หลังจาก 5 ครั้งไม่สำเร็จ จะหยุดและแสดงข้อความให้ต่อใหม่เอง
  • 🛑ถ้าตัดการเชื่อมต่อด้วยตนเอง จะไม่ trigger auto-reconnect

07 — Flow การใช้งานทั่วไป
  1. เปิดเบราว์เซอร์ไปที่ http://localhost:3000

    หน้า UI จะโหลดพร้อม sidebar ทางซ้าย และ terminal ทางขวา

  2. กรอก Host, Username, Port, Password

    หรือคลิกโปรไฟล์ที่บันทึกไว้เพื่อโหลดข้อมูลอัตโนมัติ

  3. กด "เชื่อมต่อ SSH"

    จะเห็น overlay แสดงสถานะ "กำลังเชื่อมต่อ..." ตามด้วย "เชื่อมต่อเสร็จสมบูรณ์"

  4. Sidebar ซ่อนตัวอัตโนมัติ — terminal ขยายเต็มหน้าจอ

    สามารถกด ▤ ที่ titlebar เพื่อเรียก sidebar กลับมาได้ตลอดเวลา

  5. ใช้งาน Terminal

    พิมพ์คำสั่งใน xterm หรือในช่อง Thai input bar ด้านล่าง กด Alt+Space สลับได้

  6. เปิด Tab ใหม่ (ถ้าต้องการ multi-session)

    กด + ที่ tab bar แล้วกรอกข้อมูล server ใหม่ได้เลย

  7. ตัดการเชื่อมต่อ

    กดปุ่ม ⏻ (Disconnect) ที่ titlebar หรือปุ่มกลม 🔴 มุมซ้ายของ terminal window

08 — ความปลอดภัย

✅ สิ่งที่ระบบทำ

  • 🔐ส่ง credentials ผ่าน WebSocket เดียวกับ HTTP server — ไม่ expose port SSH ออกนอก
  • 💾Password บันทึกใน localStorage ของ browser เท่านั้น
  • 🏠ออกแบบมาสำหรับ localhost หรือ private network

⚠️ ข้อควรระวัง

  • 🌐อย่า expose port 3000 ออก internet โดยไม่มี authentication
  • 🔒แนะนำให้ใช้ HTTPS + Nginx reverse proxy ถ้า deploy บน server
  • 🗝Password ใน localStorage ไม่ได้เข้ารหัส — ระวังบนเครื่องสาธารณะ

10 — แก้ไขปัญหาที่พบบ่อย

❌ เชื่อมต่อไม่ได้ / "ข้อผิดพลาด SSH"

  • 🔍ตรวจสอบ IP, port, username, password ว่าถูกต้อง
  • 🔥ตรวจสอบ firewall บน target server ว่าเปิด port SSH
  • 🖥ตรวจสอบว่า sshd รันอยู่บน server: systemctl status ssh

❌ ภาษาไทยไม่แสดงใน Terminal

  • 🔤ตั้งค่า locale บน server: export LANG=en_US.UTF-8
  • 📝ใช้ช่อง Thai input bar ด้านล่างแทนการพิมพ์ใน xterm โดยตรง

❌ Terminal ขนาดผิด / ข้อความเยื้อง

  • ↔กดปุ่ม ▤ (toggle sidebar) เพื่อ trigger resize
  • 🔄Refresh หน้าเว็บ — xterm-addon-fit จะคำนวณขนาดใหม่

❌ PWA ไม่ install ได้

  • 🔒PWA ต้องการ HTTPS หรือ localhost เท่านั้น
  • 🌐ใช้ Chrome หรือ Edge เพื่อประสบการณ์ PWA ที่ดีที่สุด
